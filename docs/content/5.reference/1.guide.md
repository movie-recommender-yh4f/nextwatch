## Setting Up Local Development

### Clone Repository

```bash
git clone <repo-url>
cd movie-recommender
```

### Install Dependencies

**Frontend:**

```bash
cd frontend
npm install
npm run dev
```

**Docs (optional):**

```bash
cd ../docs
npm install
npm run dev
```

## Development Workflow

### Frontend Development

Start Nuxt dev server:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

## File Organization

**Frontend**: `frontend/app/`

- `pages/` - Route pages
- `components/` - Reusable components
- `composables/` - Shared logic
- `types/` - TypeScript definitions
- `layouts/` - Layout templates

**Docs**: `docs/content/`

- `1.getting-started/` - Setup and architecture guides
- `3.frontend/` - Frontend reference
- `4.api/` - API reference

## Adding Features

### New Page

1. Create file in `frontend/app/pages/mypage.vue`
2. Implement component logic

### New Component

1. Create file in `frontend/app/components/MyComponent.vue`
2. Add props and emits for reactivity
3. Import and use in pages

### New Composable

1. Create file in `frontend/app/composables/useMyFeature.ts`
2. Export composable function
3. Import and use in components

## Environment Variables

Create `.env.local` in `frontend/`:

```text
NUXT_PUBLIC_SUPABASE_URL=your_url
NUXT_PUBLIC_SUPABASE_ANON_KEY=your_key
NUXT_TMDB_API_KEY=your_tmdb_key
```

## Git Workflow

1. Create feature branch: `git checkout -b feature/name`
2. Make changes
3. Test locally
4. Commit: `git commit -m "feat: description"`
5. Push: `git push origin feature/name`
6. Create pull request

## Common Issues

**Clear cache:**

```bash
rm -rf node_modules .nuxt
npm install
npm run dev
```

**Update dependencies:**

```bash
npm update
```

**Check for issues:**

```bash
npm run build
```
