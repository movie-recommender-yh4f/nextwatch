## Project Layout

The Movie Recommender project has two main directories:

```text
movie-recommender/
├── app/                         # Nuxt 4 full-stack app (where most work happens)
├── docs/                        # Documentation site
├── README.md
└── LICENSE
```

## App Directory

The main app is a Nuxt 4 application with Vue and Tailwind CSS. All active development happens here.

```text
app/
├── app/
│   ├── app.vue
│   ├── pages/
│   ├── components/
│   ├── composables/
│   ├── layouts/
│   ├── plugins/
│   └── types/
├── server/
│   ├── api/
│   │   ├── admin/
│   │   ├── movies/
│   │   ├── mylist/
│   │   ├── watched/
│   │   └── recommend.get.ts
│   └── utils/
├── public/
├── nuxt.config.ts
├── tsconfig.json
└── package.json
```

### Key Areas

- **Pages** (`app/pages/`) - Home, search, watched, profile, chat, mylist, reset-password
- **Components** (`app/components/`) - Reusable UI components
- **Composables** (`app/composables/`) - Shared logic for auth, watched state, My List, details, and filters
- **Server API** (`server/api/`) - Backend endpoints for recommendations, movie search/details, watched movies, My List, and admin import
- **Server Utilities** (`server/utils/`) - Auth helpers, rate limiting, recommendation logic, TMDB access, and TMDB import code

## Documentation Directory

The docs site (what you are reading now):

```text
docs/
├── content/            # Markdown pages
├── public/             # Static assets
├── nuxt.config.ts
└── package.json
```

## What to Know

- All app commands (`npm run dev`, `npm run build`, `npm run test`, `npm run typecheck`, `npm run lint`) are run from `app/`
- The app server runs on `http://localhost:3000`
- The docs site is separate and has its own `package.json`
- Server-side movie data, watched state, My List, and recommendation cache live in Supabase
