# TMDB Daily Import

`tmdb-import` is a Nitro scheduled task that downloads TMDB's daily movie export and upserts it into the `movies_index` table. A manual trigger endpoint is also available for admin use.

## How It Works

1. Builds a URL for yesterday's TMDB export (`movie_ids_MM_DD_YYYY.json.gz`) — yesterday is used because it is guaranteed to be available
2. Streams the gzipped NDJSON response through a gunzip pipe and a readline interface — the full ~100 MB file is never loaded into memory
3. Filters out rows where `adult: true` or `popularity < 0.5`
4. Collects valid rows into batches of 500 and inserts each batch via `@libsql/client` `batch()`
5. Uses `INSERT OR IGNORE` — rows already present in `movies_index` are silently skipped
6. If a stream error occurs, the entire attempt is retried up to 3 times with a 1.5 s delay between attempts

## Schedule

Runs daily at `0 8 * * *` (08:00 UTC).

## Result

Both the scheduled task and the manual endpoint return the same object:

```json
{
  "imported": 312847,
  "skipped": 14,
  "adultExcluded": 4201,
  "lowPopularityExcluded": 188932,
  "durationSeconds": "47.83"
}
```

| Field | Description |
|-------|-------------|
| `imported` | New rows added |
| `skipped` | Lines that could not be parsed or failed schema validation |
| `adultExcluded` | Rows dropped because `adult: true` |
| `lowPopularityExcluded` | Rows dropped because `popularity < 0.5` |
| `durationSeconds` | Wall-clock time for the full run, as a string with two decimal places |

## Manual Trigger

### `POST /api/admin/tmdb-import`

Runs the same task on demand and returns the result object above.

#### Headers

| Header | Description |
|--------|-------------|
| `x-admin-token` | Must match `ADMIN_API_TOKEN` in the server environment |

#### Error Codes

- `401` missing or incorrect `x-admin-token`
- `502` import failed after 3 attempts (stream or network error)

## Environment Variables

| Variable | Description |
|----------|-------------|
| `NUXT_LIBSQL_URL` | Database URL. Local: `file:../database/local.db`. Turso: `libsql://your-db.turso.io` |
| `NUXT_LIBSQL_AUTH_TOKEN` | Auth token for Turso. Omit for local SQLite. |
| `ADMIN_API_TOKEN` | Guards the manual trigger endpoint. |

To switch from local SQLite to Turso, update only these two environment variables — no code changes are needed.
