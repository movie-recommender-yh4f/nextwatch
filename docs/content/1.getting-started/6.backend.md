The `app/server/` folder contains the Nuxt server-side logic for TMDB proxying, Supabase-backed user state, recommendation generation, movie caching, and TMDB import handling.

## Architecture Overview

The server is organized around the API layer:

```text
app/server/
├── api/          # HTTP endpoints
│   ├── recommend.get.ts
│   ├── tmdb/      # TMDB proxy
│   ├── watched/   # Watched movies (auth required)
│   ├── mylist/    # My List (auth required)
│   ├── gemini/    # AI recommendations (auth required)
│   ├── movies/    # Movie search + movie details
│   └── admin/     # Admin endpoints (token required)
└── utils/        # Shared server utilities
```

## Current Supabase Model

The current implementation relies on four Supabase tables:

- `movies` stores the searchable TMDB corpus and the cached movie-detail fields
- `user_watched_movies` stores one row per `(user_id, tmdb_id)`
- `user_my_list` stores one row per user with `tmdb_ids integer[]`
- `recommendations` stores cached recommendation IDs in `tmdb_ids`

My List writes are atomic through RPC functions:

- `append_my_list(target_user_id uuid, target_tmdb_id integer)`
- `remove_my_list(target_user_id uuid, target_tmdb_id integer)`
## API Routes

│ Route                    | Method            | Auth            | Purpose                                                 |
| ------------------------ | ----------------- | --------------- | ------------------------------------------------------- |
| `/api/recommend`         | `GET`             | Bearer token    | Fetch cached recommendations as TMDB IDs                |
| `/api/tmdb/*`            | `GET`             | None            | Forward requests to TMDB with server-side key           |
| `/api/movies/search`     | `GET`             | None            | Search TMDB movies for the search page                  |
| `/api/movies/:id`        | `GET`             | None            | Fetch movie details and cache them in `movies`          |
| `/api/watched`           | `GET/POST/DELETE` | Bearer token    | Read/add/remove watched movies                          |
| `/api/mylist`            | `GET/POST/DELETE` | Bearer token    | Read/add/remove My List entries                         |
| `/api/gemini/recommend`  | `POST`            | Bearer token    | Generate fresh recommendations from watch history       |
| `/api/admin/tmdb-import` | `POST`            | `x-admin-token` | Trigger the TMDB import manually or from GitHub Actions |

::callout{type="warning"}
Protected routes require valid `Authorization: Bearer <token>` headers before any user-specific data operation.
::

## Recommendation Flow (`GET /api/recommend`)

Key behavior:

- Requires Bearer auth
- Reads watched history from Supabase
- Reads My List from Supabase
- Generates recommendations via Gemini when needed
- Stores valid recommendations in Supabase table `recommendations` as `tmdb_ids`
- Reuses cached recommendations for 7 days if watch history hash is unchanged

Query flags:

- `?refresh=true` forces regeneration
- `?getNew=true` forces regeneration and excludes prior cached recommendations from the next result set

## TMDB Proxy (`GET /api/tmdb/*`)

The proxy route forwards requests to TMDB using `Authorization: Bearer <NUXT_TMDB_API_KEY>` on the server.

It also applies per-IP rate limiting and exposes rate-limit headers:

- `X-RateLimit-Limit`
- `X-RateLimit-Remaining`
- `X-RateLimit-Reset`

## Important Server Utilities

- `server/utils/auth.ts` - bearer-token validation and current-user resolution
- `server/utils/tmdb.ts` - TMDB request wrapper with per-IP rate limiting
- `server/utils/recommendations.ts` - Gemini prompting + TMDB ID enrichment
- `server/utils/searchMovies.ts` - Supabase full-text lookup used when mapping recommendation titles back to TMDB IDs
- `server/utils/tmdb-import-runner.ts` - import job runner for TMDB exports

## Environment Variables

Required environment variables for the server:

```env
NUXT_TMDB_API_KEY=your_key_here
NUXT_GEMINI_API_KEY=your_gemini_api_key
NUXT_PUBLIC_SUPABASE_URL=your_supabase_url
NUXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
NUXT_SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
ADMIN_API_TOKEN=your_admin_token
UPSTASH_REDIS_REST_URL=your_upstash_redis_url
UPSTASH_REDIS_REST_TOKEN=your_upstash_redis_token
```
