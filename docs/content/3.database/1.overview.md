Movie Recommender uses two separate data stores - Supabase for user state and LibSQL for the movie index.

## Data Stores

| Store | Technology | Purpose |
|-------|-----------|---------|
| Supabase | PostgreSQL (hosted) | User authentication, session management, watched-movie records |
| LibSQL | SQLite (local file or Turso) | TMDB movie index, FTS5 full-text search, cached metadata |

## Why Two Stores?

- **Supabase** handles user identity and user data securely without needing to manage a database server. Row-level security enforces that users can only access their own records.
- **LibSQL** stores the local TMDB movie index. It is populated through the admin import endpoint, typically triggered by GitHub Actions schedules, using yesterday's TMDB export and filtering for non-adult movies with popularity of at least `0.25`.

## Supabase Schema

Two tables are used:

### `watched_movies`

| Column | Type | Description |
|--------|------|-------------|
| `id` | `uuid` | Primary key |
| `user_id` | `uuid` | Foreign key -> `auth.users.id` |
| `movies` | `jsonb` | Array of `{ tmdbId, title, year }` objects |
| `updated_at` | `timestamptz` | Last write timestamp |

Writes are upsert-like: one row per user. New movies are appended to the `movies` JSONB array, deduplicated by `tmdbId`.

### `recommendations`

Per-user cache for the `/api/recommend` endpoint.

| Column | Type | Description |
|--------|------|-------------|
| `user_id` | `uuid` | User ID cache key |
| `recommendations` | `jsonb` | Cached recommendation list |
| `watched_hash` | `text` | Hash of watched list used for cache validation |
| `expires_at` | `timestamptz` | Cache expiry time (7-day TTL) |

## LibSQL Schema

See [Schema Reference](/database/schema) for the full LibSQL table definitions.

In the current app, LibSQL full-text search is used for recommendation (mapping Gemini titles back to TMDB IDs), while the user-facing search route uses TMDB search.

## Environment Variables

| Variable | Description |
|----------|-------------|
| `NUXT_LIBSQL_URL` | Path to the local SQLite file (`file:../database/local.db`) or a Turso connection string (`libsql://your-db.turso.io`) |
| `NUXT_LIBSQL_AUTH_TOKEN` | Required only for Turso remote connections. Omit for local SQLite. |
| `NUXT_PUBLIC_SUPABASE_URL` | Your Supabase project URL |
| `NUXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase anonymous/public key |
