The Movie Recommender server currently exposes eight API routes:

- **Recommendations (Cached)** - Main recommendation endpoint used by the home page
- **TMDB Proxy** - Forward requests to The Movie Database
- **Watched Movies API** - Read/update user watched movies in Supabase
- **My List API** - Read/update the user's saved TMDB IDs
- **Gemini Recommendations** - AI-powered movie recommendations via Gemini
- **Movie Search** - Search TMDB movies for the search page
- **Movie Details** - Fetch movie metadata with Supabase-backed caching
- **TMDB Import (Admin)** - Populate the Supabase `movies` corpus from TMDB exports

All APIs are served from the server under `/api/`.

## Quick Reference

| API                      | Base URL                 | Auth                            | Rate Limit                     |
| ------------------------ | ------------------------ | ------------------------------- | ------------------------------ |
| Recommendations (Cached) | `/api/recommend`         | Bearer token (Supabase session) | 20 req / day per user (Gemini) |
| Movie Details            | `/api/movies/:id`        | None                            | App-controlled (cached)        |
| TMDB Proxy               | `/api/tmdb/*`            | None                            | 40 req / s per IP (Upstash)    |
| Watched Movies           | `/api/watched`           | Bearer token (Supabase session) | App-controlled                 |
| My List                  | `/api/mylist`            | Bearer token (Supabase session) | App-controlled                 |
| Gemini Recommendations   | `/api/gemini/recommend`  | Bearer token (Supabase session) | 20 req / day per user          |
| Movie Search             | `/api/movies/search`     | None                            | App-controlled                 |
| TMDB Import (Admin)      | `/api/admin/tmdb-import` | `x-admin-token` header          | App-controlled                 |

## Environment Variables

Required configuration for API functionality:

```bash
NUXT_TMDB_API_KEY=your_tmdb_api_key_here
NUXT_GEMINI_API_KEY=your_gemini_api_key_here
NUXT_PUBLIC_SUPABASE_URL=your_supabase_url
NUXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
NUXT_SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
ADMIN_API_TOKEN=your_admin_token
UPSTASH_REDIS_REST_URL=your_upstash_url_here
UPSTASH_REDIS_REST_TOKEN=your_upstash_key_here
```

## Authentication

Protected routes require a Supabase bearer token (`Authorization: Bearer ...`):

- `/api/recommend`
- `/api/watched`
- `/api/mylist`
- `/api/gemini/recommend`

Admin route requires `x-admin-token`:

- `/api/admin/tmdb-import`

## Rate Limiting

- TMDB-bound requests are limited per client IP
- Gemini-backed recommendation generation is limited per authenticated user
- Rate-limit headers are included on limited responses

## Data Notes

- `movies` is the current Supabase source for imported titles and cached movie details
- `user_watched_movies` stores watched state as one row per user/movie pair
- `user_my_list` stores saved TMDB IDs and is mutated through RPC functions
- `recommendations` caches `tmdb_ids` instead of full recommendation objects
