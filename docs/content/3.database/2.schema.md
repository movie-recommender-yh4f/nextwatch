## LibSQL Tables

The LibSQL database (`database/local.db`) contains three application tables plus a migrations tracker.

### `movies_index`

Primary store for all imported TMDB movies.

```sql
CREATE TABLE IF NOT EXISTS movies_index (
  tmdb_id        INTEGER PRIMARY KEY,
  original_title TEXT,
  popularity     REAL
);

CREATE INDEX IF NOT EXISTS idx_movies_popularity
ON movies_index(popularity DESC);
```

| Column           | Type      | Description                       |
| ---------------- | --------- | --------------------------------- |
| `tmdb_id`        | `INTEGER` | TMDB movie ID (primary key)       |
| `original_title` | `TEXT`    | Original-language title from TMDB |
| `popularity`     | `REAL`    | TMDB popularity score             |

By default, TMDB imports write rows with `INSERT OR IGNORE`, so existing rows are not overwritten during incremental runs. When the admin import endpoint is called with `fullRefreshQuery=true` or `fullRefreshQuery=1`, the import switches to `INSERT ... ON CONFLICT DO UPDATE` and refreshes `original_title` for existing `tmdb_id` values.

### `movies_fts`

FTS5 virtual table for full-text search over `original_title`.

```sql
CREATE VIRTUAL TABLE IF NOT EXISTS movies_fts
USING fts5(
  original_title,
  content='movies_index',
  content_rowid='tmdb_id'
);
```

Three triggers (`movies_ai`, `movies_ad`, `movies_au`) keep the FTS index in sync with `movies_index` on insert, delete, and update automatically. The TMDB import temporarily drops these triggers, bulk-loads data into `movies_index`, runs `INSERT INTO movies_fts(movies_fts) VALUES ('rebuild')`, and then recreates the triggers.

- Supports prefix queries (e.g. `toy stor*`)
- BM25 ranking is used for relevance scoring

### `movies_metadata`

Cache table for enriched TMDB metadata fetched on demand.

```sql
CREATE TABLE IF NOT EXISTS movies_metadata (
  tmdb_id       INTEGER PRIMARY KEY,
  title         TEXT,
  overview      TEXT,
  poster_path   TEXT,
  backdrop_path TEXT,
  release_date  TEXT,
  runtime       INTEGER,
  vote_average  REAL,
  vote_count    INTEGER,
  genres        TEXT,
  cast          TEXT,
  directors     TEXT,
  trailer_key   TEXT,
  cached_at     INTEGER
);

CREATE INDEX IF NOT EXISTS idx_metadata_cached
ON movies_metadata(cached_at);

CREATE INDEX IF NOT EXISTS idx_metadata_tmdb_cached
ON movies_metadata(tmdb_id, cached_at);
```

| Column          | Type      | Description                                |
| --------------- | --------- | ------------------------------------------ |
| `tmdb_id`       | `INTEGER` | TMDB movie ID (primary key)                |
| `title`         | `TEXT`    | Localised display title                    |
| `overview`      | `TEXT`    | Plot summary                               |
| `poster_path`   | `TEXT`    | TMDB poster path                           |
| `backdrop_path` | `TEXT`    | TMDB backdrop path                         |
| `release_date`  | `TEXT`    | Release date string                        |
| `runtime`       | `INTEGER` | Runtime in minutes                         |
| `vote_average`  | `REAL`    | TMDB average rating                        |
| `vote_count`    | `INTEGER` | Number of votes for the rating             |
| `genres`        | `TEXT`    | JSON array of genre names                  |
| `cast`          | `TEXT`    | JSON array of actor names (up to 5)        |
| `directors`     | `TEXT`    | JSON array of director names               |
| `trailer_key`   | `TEXT`    | YouTube video key for the official trailer |
| `cached_at`     | `INTEGER` | Unix timestamp of when the row was cached  |

## Supabase Tables

Managed by Supabase (PostgreSQL). The app currently uses `watched_movies` and `recommendations`.

```sql
CREATE TABLE watched_movies (
  id         UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id    UUID REFERENCES auth.users(id) NOT NULL,
  movies     JSONB NOT NULL DEFAULT '[]',
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

```sql
CREATE TABLE recommendations (
  user_id         UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  recommendations JSONB NOT NULL DEFAULT '[]',
  watched_hash    TEXT NOT NULL,
  expires_at      TIMESTAMPTZ NOT NULL
);
```

Row-level security (RLS) ensures each user can only read and write their own row.
