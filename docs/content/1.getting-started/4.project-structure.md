## Project Layout

The Movie Recommender project has two main directories:

```text
movie-recommender/
├── app/                         # Nuxt 4 full-stack app (where most work happens)
├── docs/                        # Documentation site
├── README.md
└── LICENSE
```

## App Directory

The main app is a Nuxt 4 application with Vue and Tailwind CSS. All active development happens here.

```text
app/
├── app/
│   ├── app.vue
│   ├── pages/
│   ├── components/
│   ├── composables/
│   ├── layouts/
│   ├── plugins/
|  └── types/
├── server/
│   ├── api/
|  │   ├── tmdb/        # TMDB proxy
|  │   ├── watched/     # Watched movies API
|  │   ├── mylist/      # My List API
|  │   ├── gemini/      # AI recommendations
|  │   ├── movies/      # Search + movie details
|  |  └── admin/       # Admin endpoints
|  └── utils/          # Server utilities
├── public/
├── nuxt.config.ts
├── tailwind.config.js
├── tsconfig.json
└── package.json
```

### Key Areas

- **Pages** (`app/pages/`) - Home, search, watched, profile, chat, reset-password
- **Components** (`app/components/`) - Reusable UI components
- **Composables** (`app/composables/`) - Shared logic for auth, watched state, details, and filters
- **Server API** (`server/api/`) - Backend endpoints for recommendations, TMDB proxying, watched movies, My List, movie details, and admin import
- **Server Utilities** (`server/utils/`) - Auth helpers, rate limiting, recommendation logic, Supabase-backed search, and TMDB import code

## Documentation Directory

The docs site (what you are reading now):

```text
docs/
├── content/            # Markdown pages
├── public/             # Static assets
├── nuxt.config.ts
└── package.json
```

## What to Know

- All commands (`npm run dev`, `npm install`, etc.) are run from the `app/` directory
- The server runs on `http://localhost:3000`
- The docs site is separate - it has its own `package.json` and dev server
- Server-side movie data, watched state, My List, and recommendation cache live in Supabase
