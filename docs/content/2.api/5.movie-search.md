`GET /api/movies/search` is used by the Search page to fetch movies by title. Returns lightweight result rows for list rendering - use `/api/movies/:id` for full movie details.

::callout{type="info"}
This route is still TMDB-backed. The Supabase `movies` table is currently used for recommendation title matching and movie-detail caching, not for the user-facing search page.
::

## Endpoint

```text
GET /api/movies/search?q=<query>
```

## Auth

No authentication required.

## Query Parameters

| Parameter | Type     | Required | Description       |
| --------- | -------- | -------- | ----------------- |
| `q`       | `string` | No       | Search query text |

If `q` is missing or empty after trimming, the endpoint returns an empty result set.

## Response Shape

```json
{
  "results": [
    {
      "id": 550,
      "title": "Fight Club",
      "original_title": "Fight Club",
      "poster_path": "/path.jpg",
      "release_date": "1999-10-15",
      "vote_average": 8.4,
      "genre_ids": [18]
    }
  ]
}
```

## Behavior

1. Reads `q` from the query string
2. Calls TMDB `/search/movie` through `fetchTmdb(...)`
3. Filters out low-quality rows (for example missing poster or zero rating)
4. Returns normalized lightweight movie rows for the client grid

## Rate Limiting

This route uses `fetchTmdb(...)`, so it inherits TMDB proxy limits:

- 40 requests per second per client IP
- Response headers: `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`

## Error Codes

- `429` TMDB rate limit exceeded
- `500` missing TMDB API key or upstream/internal failure
- `502` unexpected TMDB response format
