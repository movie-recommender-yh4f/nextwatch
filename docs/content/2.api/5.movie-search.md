# Movie Search API

`/api/movies/search` queries the SQLite FTS5 index and returns ranked movie results. No auth required.

## Endpoint

### `GET /api/movies/search?q=<query>`

Returns up to 20 matching movies.

#### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `q` | string | yes | Search query. Returns an empty array if fewer than 2 characters after trimming. |

#### Response

```json
{
  "results": [
    { "tmdb_id": 10193, "original_title": "Toy Story 3", "popularity": 59.1 },
    { "tmdb_id": 862,   "original_title": "Toy Story",   "popularity": 73.6 }
  ]
}
```

`results` is an empty array when the query is too short or no matches are found — never `null`.

#### Behavior

- Minimum 2 characters required (after trim) — shorter queries return `{ results: [] }` immediately
- FTS5 prefix matching: partial queries work (`"toy stor"` matches `"Toy Story"`)
- Each whitespace-separated token is double-quote escaped before being passed to FTS5, preventing syntax injection
- Results are ranked in order: exact title match → starts-with → contains → BM25 relevance score; popularity is the tiebreaker within each bucket
- Hard limit of 20 results

## Server Utility

`searchMovies(query)` in `frontend/server/utils/searchMovies.ts` exposes the same logic as a callable utility for use inside other server routes.

```typescript
import { searchMovies } from '~/server/utils/searchMovies'

const results = await searchMovies('inception')
// Array<{ tmdb_id: number, original_title: string, popularity: number }>
```

The Gemini recommendations route uses this utility to resolve recommended titles.

## Error Codes

- `200` always returned for valid requests — short queries and no-match cases both return an empty `results` array
