`POST /api/admin/tmdb-import` downloads TMDB's daily export and writes it into the `movies_index` table. The same endpoint is intended for local manual runs and production GitHub Actions schedules.

::callout{type="warning"}
The import is network and database intensive. Run manual imports only when needed (first setup, forced refresh, or recovery).
::

## How It Works

1. Builds a URL for yesterday's TMDB export (`movie_ids_MM_DD_YYYY.json.gz`)
2. Downloads the gzipped NDJSON export and decompresses it in the import runner
3. Parses each line as JSON and skips lines that are empty, invalid JSON, or do not match the expected TMDB export shape
4. Filters out rows where `adult: true` or `popularity < 0.25`
5. During the default incremental run, skips rows whose `id` is less than or equal to the current maximum `tmdb_id` already stored in `movies_index`
6. Collects valid rows into batches of 1500 and writes each batch with a single SQL insert
7. Uses `INSERT OR IGNORE`, so rows already present in `movies_index` are skipped during incremental imports
8. If `fullRefreshQuery=true` or `fullRefreshQuery=1` is passed, switches to `INSERT ... ON CONFLICT DO UPDATE` so existing `original_title` values can be refreshed
9. Drops the `movies_ai`, `movies_au`, and `movies_ad` FTS triggers before the import, then rebuilds `movies_fts` and recreates the triggers after the import
10. Retries batch writes up to 5 times when SQLite returns `SQLITE_BUSY`, waiting `200ms * attempt`

## Scheduling

This import is no longer scheduled through Nitro. Production scheduling should be handled in GitHub Actions by sending authenticated `POST` requests to `/api/admin/tmdb-import`.

## Result

Manual calls and GitHub Actions-triggered calls return the same object:

```json
{
  "imported": 312847,
  "skipped": 14,
  "adultExcluded": 4201,
  "lowPopularityExcluded": 188932,
  "durationSeconds": "47.83"
}
```

| Field | Description |
|-------|-------------|
| `imported` | Net increase in `movies_index` row count after the run |
| `skipped` | Rows skipped because they were invalid, failed validation, or were already covered by the current incremental watermark |
| `adultExcluded` | Rows dropped because `adult: true` |
| `lowPopularityExcluded` | Rows dropped because `popularity < 0.25` |
| `durationSeconds` | Wall-clock time for the full run, as a string with two decimal places |

## Manual Trigger

### `POST /api/admin/tmdb-import`

Runs the import on demand and returns the result object above. GitHub Actions should call this same endpoint for scheduled imports.

#### Headers

| Header | Description |
|--------|-------------|
| `x-admin-token` | Must match `ADMIN_API_TOKEN` in the server environment |

#### Query Parameters

| Parameter | Description |
|-----------|-------------|
| `fullRefreshQuery` | Optional. Use `true` or `1` to reprocess all rows from the export and update existing `original_title` values. |

#### Error Codes

- `401` missing or incorrect `x-admin-token`
- `502` import failed after the import runner exhausted its retry path or hit a stream, network, or database error

## Environment Variables

| Variable | Description |
|----------|-------------|
| `NUXT_LIBSQL_URL` | Database URL. Local: `file:../database/local.db`. Turso: `libsql://your-db.turso.io` |
| `NUXT_LIBSQL_AUTH_TOKEN` | Auth token for Turso. Omit for local SQLite. |
| `ADMIN_API_TOKEN` | Guards the manual trigger endpoint. |

To switch from local SQLite to Turso, update only these two environment variables and keep the GitHub Actions or manual API call pattern the same.
