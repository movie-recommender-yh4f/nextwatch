# Backend Server

The `frontend/server/` folder contains Nuxt server-side logic for TMDB proxying, authenticated watched-movies persistence, AI recommendations via Gemini, full-text movie search, and scheduled TMDB data imports.

## Architecture Overview

The server is currently organized around the API layer:

```text
frontend/server/
├── api/          # HTTP endpoints
│   ├── tmdb/     # TMDB proxy
│   ├── watched/  # Watched movies (auth required)
│   ├── gemini/   # AI recommendations (auth required)
│   ├── movies/   # Movie search
│   └── admin/    # Admin endpoints (token required)
├── tasks/        # Nitro scheduled tasks
└── utils/        # Shared server utilities
```

## API Routes (`api/`)

### TMDB Proxy

**File:** `tmdb/[...path].ts`

An endpoint that forwards requests to The Movie Database (TMDB) API.

- **Purpose:** Centralize TMDB API calls and manage credentials securely
- **Endpoint:** `/api/tmdb/*`
- **Key Features:**
  - Adds API key from environment variables
  - Sets default language to `en-US`
  - Handles JSON responses
  - Error handling with status code forwarding

### Watched Movies

**File:** `watched/index.ts`

Authenticated endpoint backed by Supabase table `watched_movies`.

- **Purpose:** Persist and load per-user watched movies
- **Endpoint:** `/api/watched`
- **Methods:**
  - `GET` - return current user's watched movies
  - `POST` - append movie to watched list (deduplicated by `tmdbId`)
- **Key Features:**
  - Bearer-token auth with Supabase `auth.getUser(...)`
  - JSONB array storage (`movies`)
  - Upsert-like behavior (insert first row or update existing row)
  - `updated_at` refresh on writes

### Gemini Recommendations

**File:** `gemini/recommend.post.ts`

AI-powered recommendations backed by Google Gemini.

- **Purpose:** Generate personalised movie recommendations from watch history
- **Endpoint:** `POST /api/gemini/recommend`
- **Key Features:**
  - Requires Bearer token auth (Supabase JWT)
  - Merges Supabase watch history with optional local movies from request body
  - Resolves each recommended title to a TMDB ID via the local SQLite
  - Returns up to 20 recommendations with `name`, `originalName`, `year`, and `tmdbId`

### Movie Search

**File:** `movies/search.get.ts`

Full-text search over the local SQLite movies index.

- **Purpose:** Search movie titles without hitting the TMDB API
- **Endpoint:** `GET /api/movies/search?q=<query>`
- **Key Features:**
  - No auth required
  - Minimum 2 characters — shorter queries return `{ results: [] }` immediately
  - Hard limit of 20 results
  - Available as a shared utility: `searchMovies(query)` in `server/utils/searchMovies.ts`

### TMDB Daily Import (Admin)

**File:** `admin/tmdb-import.post.ts`

Manual trigger for the TMDB data import task.

- **Purpose:** On-demand rerun of the scheduled TMDB import
- **Endpoint:** `POST /api/admin/tmdb-import`
- **Key Features:**
  - Inserts into `movies_index` (SQLite), filtering `adult: true` and `popularity < 0.5`
  - Runs automatically via Nitro scheduled task every day at 08:00 UTC

## Environment Variables

Required environment variables for the server:

```env
NUXT_TMDB_API_KEY=your_key_here
NUXT_GEMINI_API_KEY=your_gemini_api_key
NUXT_LIBSQL_URL=file:../database/local.db
NUXT_LIBSQL_AUTH_TOKEN=               # Only needed for Turso (remote LibSQL)
NUXT_PUBLIC_SUPABASE_URL=your_supabase_url
NUXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
ADMIN_API_TOKEN=your_admin_token
```

## Request Flow Example

**Fetching movie details through the proxy:**

1. Client requests `/api/tmdb/movie/550`
2. Route handler forwards the request to TMDB
3. The server injects `NUXT_TMDB_API_KEY`
4. JSON response is returned to the frontend

## Security Model

| Route | Authentication | Rate Limit |
|-------|----------------|------------|
| `/api/tmdb/*` | API Key (env) | TMDB limits |
| `/api/watched` | Bearer token (Supabase session) | App-controlled |
