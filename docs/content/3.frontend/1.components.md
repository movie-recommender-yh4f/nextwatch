## Pages

Vue pages in `app/pages/`:

### index.vue

Main landing page. Shows:

- Popular titles from TMDB
- Swipe-style browsing cards
- Quick access to movie details

### login.vue

Authentication page. Handles:

- Email/password login
- Sign up form
- Forgot password link

### profile.vue

User profile page. Displays:

- User information
- Watch statistics
- Static preference placeholders
- Logout action for testing auth/session behavior

### search.vue

::note
Search currently uses the popular-movies feed and local filtering.
::

Movie search and browse. Features:

- Search by title
- Filter by genre name
- Display a simple grid of matches

### watched.vue

User's watch history. Shows:

- Movies the user has marked as watched
- Details fetched from TMDB
- Quick access to movie details

### reset-password.vue

Password recovery flow:

- Email verification
- New password creation
- Confirmation

## Composables

Shared logic in `app/composables/`:

### useAuth.ts

Authentication management:

```ts
const { user, login, logout, signup, resetPassword } = useAuth()
```

### useMovies.ts

Movie data fetching:

```ts
const {
  getPopularMovies,
  getMovieDetails,
  markAsWatched,
  syncWatchedMoviesFromSupabase
} = useMovies()
```

Watched-movie behavior:

- `syncWatchedMoviesFromSupabase(token?)`
  - Loads watched movie IDs from `GET /api/watched`
  - Hydrates `watchedMovies` state when session exists
  - Clears local watched state when no session is available
- `markAsWatched(movie)`
  - Returns `'ok' | 'unauthorized' | 'error'`
  - On authenticated users: updates local state and calls `POST /api/watched`
  - On unauthenticated users: caller can redirect to login

### useSupabase.ts

Supabase access:

```ts
const supabase = useSupabase()
```

### Auth and watched sync

`useAuth.ts` calls watched sync during:

- `initialize()` (session restore on app startup)
- `onAuthStateChange` callback
- `login()` and `signup()`
- `logout()` (clears local watched state via sync path)
