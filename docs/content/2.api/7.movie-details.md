The Movie Details endpoint fetches enriched metadata for a single movie by its TMDB ID. Results are cached server-side in the Supabase `movies` table for about **3 months** to avoid redundant TMDB API calls, and TMDB `404` misses are briefly negative-cached in Redis to avoid repeated lookups for nonexistent IDs.

The route is also rate-limited before any TMDB request is made. Authenticated users can make up to **20 requests per second**, and guests can make up to **5 requests per second**.

## Endpoint

```text
GET /api/movies/:id
```

## How It Works

1. Client requests `/api/movies/:id` with a TMDB movie ID
2. Server checks `movies` for a cached row
3. **Positive cache hit** - returns the cached record immediately, no TMDB call made
4. **Negative cache hit** - returns `404` immediately for a recently confirmed missing TMDB ID
5. **Cache miss or stale/incomplete row** - checks the miss-only limiter, fetches from TMDB (`/movie/:id?append_to_response=credits,videos`), upserts the movie row in Supabase, and returns the normalized response

## Example Usage

```js
const response = await fetch('/api/movies/550')
const movie = await response.json()
```

## Path Parameter

| Parameter | Type     | Description                      |
| --------- | -------- | -------------------------------- |
| `id`      | `number` | TMDB movie ID (positive integer) |

Returns `400` if the ID is missing or not a positive integer.

## Response Format

```json
{
  "id": 550,
  "title": "Fight Club",
  "poster": "https://image.tmdb.org/t/p/w500/path/to/poster.jpg",
  "rating": 8.4,
  "year": 1999,
  "duration": "2h 19m",
  "runtime": 139,
  "genres": ["Drama", "Thriller"],
  "actors": ["Brad Pitt", "Edward Norton", "Helena Bonham Carter", "Meat Loaf", "Zach Grenier"],
  "directors": ["David Fincher"],
  "description": "A ticking-time-bomb insomniac...",
  "trailer": "https://www.youtube.com/watch?v=SUXWAEX2jlg"
}
```

| Field         | Type       | Notes                                                 |
| ------------- | ---------- | ----------------------------------------------------- |
| `id`          | `number`   | TMDB ID                                               |
| `title`       | `string`   | Movie title                                           |
| `poster`      | `string`   | Full image URL (`w500`). Empty string if no poster    |
| `rating`      | `number`   | `vote_average` rounded to 1 decimal place             |
| `year`        | `number`   | Release year parsed from `release_date`               |
| `duration`    | `string`   | Formatted as `2h 19m`. `"N/A"` if runtime is unknown  |
| `runtime`     | `number`   | Raw runtime in minutes, or `null` when unavailable    |
| `genres`      | `string[]` | Genre names                                           |
| `actors`      | `string[]` | Top 5 cast members                                    |
| `directors`   | `string[]` | Director names                                        |
| `description` | `string`   | Movie overview                                        |
| `trailer`     | `string`   | YouTube link or `null`                                |

## Caching Details

| Property          | Value                                                                        |
| ----------------- | ---------------------------------------------------------------------------- |
| Positive cache store | Supabase `movies` table                                                   |
| Positive TTL      | ~3 months (90 days)                                                          |
| Positive cache key | `tmdb_id`                                                                   |
| Negative cache store | Upstash Redis                                                             |
| Negative TTL      | 24 hours                                                                     |
| Trailer selection | Most recent official YouTube trailer (`type = "Trailer"`, `official = true`) |

Cache freshness is checked via the `cached_at` timestamp column. A record is also refreshed when required detail fields are still empty.

## Rate Limiting

`GET /api/movies/:id` uses a fixed-window limiter in `server/utils/movies/details-rate-limit.ts`.

| Identity | Limit | Window | Key |
| -------- | ----- | ------ | --- |
| Authenticated user | 20 requests | 1 second | `movie-details:user:<userId>` |
| Guest | 5 requests | 1 second | `movie-details:guest:<ip>` |
| Authenticated uncached miss | 100 requests | 1 day | `movie-details-miss:user:<userId>` |
| Guest uncached miss | 20 requests | 1 day | `movie-details-miss:guest:<ip>` |

The endpoint includes the standard `X-RateLimit-*` headers and the movie-details-specific `X-Movie-Details-RateLimit-*` headers on every response. Uncached anonymous misses use the durable miss budget only when `x-vercel-forwarded-for` is available.

## Error Handling

| Status | Cause                                     |
| ------ | ----------------------------------------- |
| `400`  | `id` is missing or not a positive integer |
| `404`  | TMDB returned no movie for that ID        |
| `500`  | TMDB API or Supabase failure              |
