# Schema Reference

The canonical schema lives in [`database/schema.sql`](../../../database/schema.sql). Apply it once to create all tables, triggers, and indexes.

## LibSQL Tables

The LibSQL database (`database/local.db`) contains three application tables plus a migrations tracker.

### `movies_index`

Primary store for all imported TMDB movies.

```sql
CREATE TABLE IF NOT EXISTS movies_index (
  tmdb_id        INTEGER PRIMARY KEY,
  original_title TEXT,
  popularity     REAL
);

CREATE INDEX IF NOT EXISTS idx_movies_popularity
ON movies_index(popularity DESC);
```

| Column | Type | Description |
|--------|------|-------------|
| `tmdb_id` | `INTEGER` | TMDB movie ID (primary key) |
| `original_title` | `TEXT` | Original-language title from TMDB |
| `popularity` | `REAL` | TMDB popularity score |

Rows are inserted via `INSERT OR IGNORE` — existing rows are never overwritten.

### `movies_fts`

FTS5 virtual table for full-text search over `original_title`.

```sql
CREATE VIRTUAL TABLE IF NOT EXISTS movies_fts
USING fts5(
  original_title,
  content='movies_index',
  content_rowid='tmdb_id'
);
```

Three triggers (`movies_ai`, `movies_ad`, `movies_au`) keep the FTS index in sync with `movies_index` on insert, delete, and update automatically.

- Supports prefix queries (e.g. `toy stor*`)
- BM25 ranking is used for relevance scoring

### `movies_metadata`

Cache table for enriched TMDB metadata fetched on demand.

```sql
CREATE TABLE IF NOT EXISTS movies_metadata (
  tmdb_id       INTEGER PRIMARY KEY,
  title         TEXT,
  overview      TEXT,
  poster_path   TEXT,
  backdrop_path TEXT,
  release_date  TEXT,
  runtime       INTEGER,
  vote_average  REAL,
  json_data     TEXT,
  cached_at     INTEGER
);

CREATE INDEX IF NOT EXISTS idx_metadata_cached
ON movies_metadata(cached_at);
```

| Column | Type | Description |
|--------|------|-------------|
| `tmdb_id` | `INTEGER` | TMDB movie ID (primary key) |
| `title` | `TEXT` | Localised display title |
| `overview` | `TEXT` | Plot summary |
| `poster_path` | `TEXT` | TMDB poster path |
| `backdrop_path` | `TEXT` | TMDB backdrop path |
| `release_date` | `TEXT` | Release date string |
| `runtime` | `INTEGER` | Runtime in minutes |
| `vote_average` | `REAL` | TMDB average rating |
| `json_data` | `TEXT` | Full TMDB response JSON blob |
| `cached_at` | `INTEGER` | Unix timestamp of when the row was cached |

### `_schema_migrations`

Tracks applied migrations for the Turso/LibSQL migration path.

```sql
CREATE TABLE IF NOT EXISTS _schema_migrations (
  id         INTEGER PRIMARY KEY,
  version    TEXT NOT NULL UNIQUE,
  applied_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

## Supabase Tables

Managed by Supabase (PostgreSQL). The single application table is `watched_movies`.

```sql
CREATE TABLE watched_movies (
  id         UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id    UUID REFERENCES auth.users(id) NOT NULL,
  movies     JSONB NOT NULL DEFAULT '[]',
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

Row-level security (RLS) ensures each user can only read and write their own row.
