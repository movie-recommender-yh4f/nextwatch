# Local Database Setup

The LibSQL database is a plain SQLite file at `database/local.db`. It does **not** exist in the repository — you create and populate it yourself.

## Prerequisites

- Node.js v18+
- The frontend dev server running (`npm run dev` inside `frontend/`)
- `ADMIN_API_TOKEN` set in `frontend/.env.local`
- `NUXT_LIBSQL_URL=file:../database/local.db` set in `frontend/.env.local`

## Step 1 — Apply the Schema

Run the schema SQL against the local database file once to create all tables, triggers, and indexes.

The simplest way is via the SQLite CLI:

```bash
sqlite3 database/local.db < database/schema.sql
```

::callout
After this step `database/local.db` exists and has the correct structure, but all tables are empty.
::

## Step 2 — Populate the Movie Index

Trigger the TMDB daily import to download and insert all movies. This downloads yesterday's TMDB export file and upserts it into `movies_index`.

With the dev server running:

```bash
curl -X POST http://localhost:3000/api/admin/tmdb-import \
  -H "x-admin-token: $ADMIN_API_TOKEN"
```

Or with PowerShell:

```powershell
Invoke-RestMethod -Method Post `
  -Uri "http://localhost:3000/api/admin/tmdb-import" `
  -Headers @{ "x-admin-token" = $env:ADMIN_API_TOKEN }
```

A successful response looks like:

```json
{
  "imported": 312847,
  "skipped": 14,
  "adultExcluded": 4201,
  "lowPopularityExcluded": 188932,
  "durationSeconds": "360"
}
```

Once this completes, `movies_index` and `movies_fts` are populated and movie search is fully operational.

::callout{type="info"}
The import runs automatically every day at 08:00 UTC once the app is deployed. The manual trigger above is only needed for the initial local setup or to force a reimport.
::

## Step 3 — Verify

Check that rows exist in the index:

```bash
sqlite3 database/local.db "SELECT COUNT(*) FROM movies_index;"
```

And that the FTS index is working:

```bash
sqlite3 database/local.db \
  "SELECT tmdb_id, original_title FROM movies_fts WHERE movies_fts MATCH 'inception' LIMIT 5;"
```

Or via the API:

```bash
curl "http://localhost:3000/api/movies/search?q=inception"
```

## Switching to Turso (Remote LibSQL)

To use a hosted Turso database instead of the local file, update two env vars — no code changes are needed:

```bash
NUXT_LIBSQL_URL=libsql://your-db.turso.io
NUXT_LIBSQL_AUTH_TOKEN=your_turso_auth_token
```

Then apply the schema via the Turso CLI:

```bash
turso db shell your-db < database/schema.sql
```

And trigger the import the same way as Step 2.

## Troubleshooting

### `no such table: movies_index`

The schema has not been applied. Run Step 1.

### Import returns `502`

The TMDB export URL for yesterday was not available. Wait a few minutes and retry — the import has built-in retry logic (3 attempts, 1.5 s apart), but a 502 means all attempts failed. Check the server logs for the upstream error.

### `NUXT_LIBSQL_URL is not set`

Add `NUXT_LIBSQL_URL=file:../database/local.db` to `frontend/.env.local` and restart the dev server.

### `database/local.db` is gitignored

This is intentional — the database file is large and regenerable. The `.gitignore` in `database/` excludes `*.db`.
