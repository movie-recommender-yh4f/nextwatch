# API Reference

The Movie Recommender server currently exposes five APIs:

- **TMDB Proxy** - Forward requests to The Movie Database
- **Watched Movies API** - Read/update user watched movies in Supabase
- **Gemini Recommendations** - AI-powered movie recommendations via Google Gemini
- **Movie Search** - Full-text search over the local SQLite movie index
- **TMDB Import (Admin)** - Manually trigger the scheduled TMDB data import

All APIs are served from the frontend server under `/api/`.

## Quick Reference

| API | Base URL | Auth | Rate Limit |
|-----|----------|------|-----------|
| TMDB Proxy | `/api/tmdb/*` | API Key (env) | TMDB limits |
| Watched Movies | `/api/watched` | Bearer token (Supabase session) | App-controlled |
| Gemini Recommendations | `/api/gemini/recommend` | Bearer token (Supabase session) | Gemini limits |
| Movie Search | `/api/movies/search` | None | App-controlled |
| TMDB Import (Admin) | `/api/admin/tmdb-import` | `x-admin-token` header | App-controlled |

## Environment Variables

Required configuration for API functionality:

```bash
NUXT_TMDB_API_KEY=your_tmdb_api_key_here
NUXT_GEMINI_API_KEY=your_gemini_api_key_here
NUXT_LIBSQL_URL=file:../database/local.db
NUXT_PUBLIC_SUPABASE_URL=your_supabase_url
NUXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
ADMIN_API_TOKEN=your_admin_token
```

## Response Format

The proxy returns JSON and forwards TMDB response bodies as-is.

## Authentication Methods

### API Key (TMDB Proxy)

Automatically injected from the `NUXT_TMDB_API_KEY` environment variable.

### Bearer Token (Watched Movies API)

The client sends a Supabase access token (`Authorization: Bearer ...`) and the server resolves the user via Supabase auth.

## CORS & Security

- **TMDB Proxy**: Securely forwards requests with the API key from the server
- **Watched Movies API**: Requires user auth token and scopes writes/reads to `user_id`
