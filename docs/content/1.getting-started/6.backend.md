# Backend Server

The `frontend/server/` folder contains the current Nuxt server-side logic. Right now its job is intentionally small: proxy TMDB requests so the frontend can fetch movie data without exposing the API key directly.

## Architecture Overview

The server is currently organized around the API layer:

```text
frontend/server/
`-- api/          # HTTP endpoints
```

## API Routes (`api/`)

### TMDB Proxy

**File:** `tmdb/[...path].ts`

An endpoint that forwards requests to The Movie Database (TMDB) API.

- **Purpose:** Centralize TMDB API calls and manage credentials securely
- **Endpoint:** `/api/tmdb/*`
- **Key Features:**
  - Adds API key from environment variables
  - Sets default language to `en-US`
  - Handles JSON responses
  - Error handling with status code forwarding

## Current Recommendation Status

The previous similarity-matrix endpoints were intentionally removed. Recommendation delivery is not implemented in the current server and will be reintroduced later as a separate external-provider integration.

## Environment Variables

Required environment variables for the server:

```env
NUXT_TMDB_API_KEY=your_key_here
```

## Request Flow Example

**Fetching movie details through the proxy:**

1. Client requests `/api/tmdb/movie/550`
2. Route handler forwards the request to TMDB
3. The server injects `NUXT_TMDB_API_KEY`
4. JSON response is returned to the frontend

## Security Model

| Route | Authentication | Rate Limit |
|-------|----------------|------------|
| `/api/tmdb/*` | API Key (env) | TMDB limits |
